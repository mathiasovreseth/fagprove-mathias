    Introduction

    API documentation for Mathias' examination

    Authentication

    Login

    Example input:

    {
    "email": "kristoffer@munikum.no",
    "password": "testing"
}

    Example output:

    {
    "username": "kristoffer@munikum.no",
    "roles": [
        "ROLE_ADMIN"
    ],
    "token_type": "Bearer",
    "access_token": "eyJhbG...",
    "expires_in": 36000,
    "refresh_token": "eyJhbG..."
}

    Login with username and password.

    HTTP Request

    POST http://localhost:8080/auth/login

    Parameters

    Parameter Description
    email Your email
    password Your password

    API Usage

    Include your access token in the authorization header.

    Authorization: Bearer xxx

    You must replace xxx with your access token.

    Creates a new person

    HTTP Request

    POST http://localhost:8080/person/create

    Parameters

    Parameter Optional Description
    email False Persons email
    name False Persons name
    password False Persons password
    role False Persons access level. Can be ROLE_ADMIN or ROLE_MANAGER or ROLE_USER
    personType False Persons type. Can be EXAMINATOR or CANDIDATE or ADMIN
    jobRole True Persons job role
    phoneNumber True Persons phone number
    company True Persons company
    region True Region of persons company
    registrationReceived True If person is registered as ready for examination
    committees True Committees the user is member of
    Needs ROLE_ADMIN or ROLE_MANAGER access level.

    Updates an existing person

    HTTP Request

    POST http://localhost:8080/person/update

    Parameters

    Parameter Optional Description
    id False ID of person to update
    email False Persons email
    name False Persons name
    password False Persons password
    role False Persons access level. Can be ROLE_ADMIN or ROLE_MANAGER or ROLE_USER
    personType False Persons type. Can be EXAMINATOR or CANDIDATE or ADMIN
    jobRole True Persons job role
    phoneNumber True Persons phone number
    company True Persons company
    region True Region of persons company
    registrationReceived True If person is registered as ready for examination
    committees True Committees the user is member of
    Needs ROLE_ADMIN or ROLE_MANAGER access level.

    Set Person as Busy

    Example input:

    {
    "person": 3,
    "from": "2020-04-01T00:00:00Z",
    "to": "2020-04-04T00:00:00Z"
}

    Set person as busy for date range

    HTTP Request

    POST http://localhost:8080/person/setBusy

    Parameters

    Parameter Optional Description
    person False ID of person to set as busy
    from False Start date of busy period
    to False End date of busy period
    Needs ROLE_ADMIN or ROLE_MANAGER access level. Users with ROLE_MANAGER access level can only set busy status for themselves.
